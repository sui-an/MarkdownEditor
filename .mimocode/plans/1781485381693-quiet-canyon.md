# Tree-sitter 语法高亮集成方案

## 目标

为 MarkdownEditor 添加完整的语法高亮功能：
1. **Markdown 语法高亮** — 使用 tree-sitter-markdown 解析器，支持所有 markdown 语法元素（标题、粗体、斜体、链接、图片、引用、代码块、任务列表、表格、水平线等）
2. **代码块语言高亮** — 在 fenced code block 内根据语言（Python, JS, Swift 等）进行语法高亮
3. **增量解析** — 仅重新解析编辑区域，保持大文件性能

## 技术选型

| 组件 | 选择 | 说明 |
|------|------|------|
| 构建系统 | SPM | 从 swiftc 迁移到 Swift Package Manager |
| 树解析器 | Neon + SwiftTreeSitter | ChimeHQ 的 Neon 提供 NSTextView 集成 |
| Markdown 解析器 | tree-sitter-markdown | MDeiml/tree-sitter-markdown |
| 语言解析器 | 按需添加 | Python, JS, TS, Swift, Go, Rust, Java, C, C++ 等 |

### Phase 2: Tree-sitter 核心集成

**目标**: 创建 TreeSitterManager 管理解析器和语言配置

**新增文件**:
- `Sources/Services/TreeSitter/TreeSitterManager.swift` — 管理所有语言解析器
- `Sources/Services/TreeSitter/HighlightTheme.swift` — token 类型到颜色的映射
- `Sources/Services/TreeSitter/LanguageProvider.swift` — 语言识别和配置

**TreeSitterManager 职责**:
1. 初始化所有语言解析器
2. 管理 LanguageConfiguration
3. 处理语言注入（code blocks 内的语言识别）
4. 提供 highlights 查询接口

**语言注入逻辑**:
```swift
// Markdown 中的代码块检测
// ```python → 使用 Python 解析器
// ```swift → 使用 Swift 解析器
// 无语言标签 → 使用纯文本（无高亮）
```

### Phase 3: MarkdownTextStorage 改造

**目标**: 将正则高亮替换为 tree-sitter 高亮

**变更文件**:
- `Sources/Views/Editor/MarkdownTextStorage.swift` — 重写高亮逻辑

**改造方案**:

1. **保留 NSTextStorage 子类** — Neon 的 `TextViewHighlighter` 可以直接使用
2. **集成 TreeSitterClient** — 在 `processEditing()` 中触发 tree-sitter 解析
3. **应用高亮** — 将 tree-sitter 查询结果映射到 NSAttributedString 属性

**关键代码结构**:
```swift
final class MarkdownTextStorage: NSTextStorage {
    private let backingStore = NSMutableAttributedString()
    private var treeSitterClient: TreeSitterClient?
    private var highlightTheme: HighlightTheme
    
    override func processEditing() {
        super.processEditing()
        // 使用 tree-sitter 增量解析
        // 仅重新高亮变化区域
    }
}
```

### Phase 4: Markdown 语法增强

**目标**: 支持所有 markdown 语法元素

**tree-sitter-markdown 支持的元素**:
- ATX headings (`# Heading`)
- Setext headings (`Heading\n===`)
- Thematic breaks (`---`)
- Blockquotes (`> text`)
- Fenced code blocks (```` ```lang ````)
- Indented code blocks
- Lists (ordered/unordered)
- Task lists (`- [x]`)
- Tables
- Links, images
- Bold, italic, strikethrough
- Inline code
- HTML blocks

**高亮方案**:
- 每种元素映射到独特的颜色
- 支持 light/dark 主题切换
- 代码块内的语言由注入系统处理

### Phase 5: 代码块语言高亮

**目标**: 在代码块内根据语言进行语法高亮

**支持的语言（初始）**:
1. Python
2. JavaScript
3. TypeScript
4. Swift
5. Go
6. Rust

**语言识别**:
- 从 fenced code block 的语言标签识别
- 支持别名（js → javascript, ts → typescript）

## 性能优化

1. **增量解析** — Tree-sitter 仅重新解析变化区域
2. **后台解析** — 大文件在后台队列解析，避免阻塞主线程
3. **可见区域优先** — 优先高亮可见区域，后台处理其余部分
4. **解析缓存** — 缓存解析结果，避免重复解析
5. **延迟高亮** — 编辑时延迟 100ms 再触发高亮（类似当前实现）

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SPM 迁移导致构建失败 | 高 | 逐步迁移，保持 swiftc 作为后备 |
| Tree-sitter 解析器内存占用 | 中 | 仅加载常用语言，按需加载 |
| 大文件解析性能 | 中 | 后台解析 + 可见区域优先 |
| Neon API 不稳定 | 中 | 使用 SwiftTreeSitter 直接集成作为后备 |
| 语言解析器兼容性 | 低 | 测试主流语言版本 |
